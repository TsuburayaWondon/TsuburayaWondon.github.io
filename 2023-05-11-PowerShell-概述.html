什么是 PowerShell?

PowerShell 是一种跨平台的任务自动化解决方案，包括命令行 shell、脚本语言和配置管理框架。 PowerShell 在 Windows、Linux 和 macOS 上运行。下面主要介绍命令行 shell 和脚本语言。

命令行 shell

  • shell是什么？
    shell 是一种交互式命令行解释器（输入立刻就可得到输出），它是操作系统与用户之间的接口，用户可以通过 Shell 执行各种操作系统命令，也可以编写 shell 脚本进行自动化操作。在 Linux、Unix 和 macOS 等类 Unix 操作系统中，shell 是一个基本的组成部分，常见的 Shell 包括 Bash、Zsh、Fish 等。在 Windows 系统中，也有自带的命令行解释器 cmd.exe 和 PowerShell，它们同样也是 shell 的一种实现。

  • PowerShell 是一种新式命令 shell，其中包括其它常用 shell 的最佳功能。如下：

    • 可靠的命令行历史记录
    • Tab 自动补全和命令预测
    • 支持命令和参数别名
    • 用于链接命令的管道
    • 控制台内帮助系统，类似于 Linux/Unix man 页面

脚本语言

作为一种脚本语言，PowerShell 通常用于自动执行系统管理。

  • PowerShell 脚本语言包含以下功能：
    • 可通过函数、类、脚本和模块进行扩展
    • 便于输出的可扩展格式系统
    • 用于创建动态类型的可扩展类型系统
    • 对常用数据格式（例如 CSV、JSON 和 XML）的内置支持

什么是 PowerShell 命令？

PowerShell 命令被称为 cmdlet（读作 command-let）（以后我们会很少提到“PowerShell 命令”这一说法，而是用cmdlet）

  • cmdlet 命名

    • PowerShell 使用“动词-名词”名称对来命名 cmdlet
      例如，PowerShell 中包含的 Get-Command cmdlet 用于获取在命令行界面中注册的所有 cmdlet。 动词标识 cmdlet 执行的操作，名词标识该 cmdlet 执行其操作的所在资源。这种命名约定便于人们了解 cmdlet 的作用。 还能让你更轻松地找到所查找的命令。 当你查找要使用的 cmdlet 时，可以根据动词或名词进行筛选。

    • 使用完整的英文单词
      为了确保易于理解，PowerShell使用完整的英文单词而不是缩写。例如，使用 Parameter（参数） 而不是 Param
