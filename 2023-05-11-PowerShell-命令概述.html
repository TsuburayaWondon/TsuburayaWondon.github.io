参数项

  • command-name
    命令名称。命令总是以命令名称开头，命令名称可以是标准的命令名也可以是它的别名

  • Parameter
    参数。命令的参数是确定命令的作用的选项。

    • Parameter Name
      参数名。前面带上连接短线“-”。（这和其它 shell 稍有不同，例如在 bash shell 中，带“-”的我们一般称之为选项，参数是另外的概念。但是在 PowerShell 中，我们全部称之为“参数”）

    • Parameter Value
      参数值。参数值是参数采用的输入。（由于Windows PowerShell基于 Microsoft .NET Framework，因此参数值按 .NET 类型在语法关系图中表示）

    • Required…
      左右尖括号“<>”。表示必选。

    • Optional…
      左右方括号“[]”。表示可选。

  • Switch Parameter
    开关参数。不带参数值的参数。

参数集

命令的参数列在参数集中。参数集类似于语法关系图的段落。简单地说，参数集是一个命令可能同时有的参数的集合。某些命令的参数集只有一个，但是多数命令的参数集有多个。

  • 参数集仅包含可在命令中一起使用的参数。 如果参数不能在同一命令中使用，它们将显示在单独的参数集中。

  • 每个 cmdlet 都有一个默认参数集。如果未指定参数集唯一的参数，则使用默认参数集。

  • 在每个参数集中，参数按位置顺序显示。一般来说参数的顺序是任意的。仅当省略可选参数名称时，命令中的参数顺序才重要。省略参数名称时，PowerShell 按位置和类型将值分配给参数。

  • 尖括号 <> 表示占位符文本
    不要直接在命令中键入尖括号或占位符文本（它只是一个通配符哦😉），请将它替换为它描述的具体的项。
    尖括号用于标识参数采用的值的 .NET 类型。例如，若要使用 cmdlet 的 New-Alias -Name 参数，请将字符串<string>替换为一个字符串，该字符串是一个单词或一组用引号括起来的单词。

  • [] 方括号指示可选项。参数及其值可以是可选的，或者所需参数的名称可以是可选的。
